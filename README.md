# Hotel Booking QA Engineering Project

End-to-end QA project based on a hotel booking web application.

This project demonstrates a complete QA workflow, including manual testing, API testing, UI automation, accessibility testing, performance testing, CI/CD, and AI-assisted QA documentation.

## Application Under Test

**Website:** https://automationintesting.online/

The application simulates a hotel booking platform where users can view rooms, make reservations, submit contact forms, and interact with different UI components.

## Project Goals

- Analyze the application from a QA perspective.
- Define functional scope, risks, and acceptance criteria.
- Design manual test cases based on business flows.
- Report defects with clear evidence.
- Automate critical user flows using Playwright.
- Validate API endpoints using Postman.
- Run performance checks using k6.
- Add accessibility testing using WCAG-based checks.
- Integrate automated tests into GitHub Actions.
- Explore AI-assisted workflows for QA documentation and bug analysis.

## QA Areas Covered

- Manual Testing
- Functional Testing
- Regression Testing
- API Testing
- UI Automation
- Accessibility Testing
- Performance Testing
- CI/CD
- AI-assisted QA workflows
