# Acceptance Criteria

## US-001: View Available Rooms

- The homepage displays available rooms.
- Each room displays relevant information such as name, description, price, and image.
- Room information should be visible without layout issues.
- Users should be able to identify available booking options.

## US-002: View Room Details

- Users can access detailed information for a selected room.
- Room details should include name, description, price, and room features where available.
- Images should load correctly.
- Room information should be consistent across the application.

## US-003: Select Booking Dates

- Users can select a valid check-in date.
- Users can select a valid check-out date.
- Check-out date should be after check-in date.
- Invalid date combinations should not be accepted.
- Date picker should be usable and clear.

## US-004: Submit Booking Information

- Users can enter required booking information.
- Required fields should include first name, last name, email, phone, and booking dates where applicable.
- Valid information should allow booking submission.
- Invalid or missing information should prevent booking submission.

## US-005: Receive Booking Confirmation

- After submitting a valid booking, the user should receive a clear confirmation message.
- The confirmation should indicate that the booking was successfully created.
- The user should not see a success message if required data is missing or invalid.

## US-006: Validate Required Booking Fields

- Missing required fields should trigger validation messages.
- Invalid email format should be rejected.
- Invalid phone format should be rejected where applicable.
- Validation messages should be clear and located near the relevant fields.
- The user should be able to correct the form and resubmit it.

## US-007: Send Contact Message

- Users can enter name, email, phone, subject, and message.
- Valid contact form data should be submitted successfully.
- A confirmation message should be displayed after successful submission.

## US-008: Validate Contact Form Fields

- Empty required fields should trigger validation messages.
- Invalid email format should trigger an error.
- Message length rules should be enforced where applicable.
- The form should not submit invalid data.

## US-009: Navigate Through the Website

- Navigation links should direct users to the correct sections.
- Buttons and links should be clickable.
- Navigation should work consistently across supported screen sizes.
- There should be no broken links in the main flow.

## US-010: Use the Website on Different Screen Sizes

- Content should remain readable on desktop and mobile widths.
- Forms should be usable on smaller screens.
- Buttons should remain visible and clickable.
- No critical content should overlap or disappear.

## US-011: Understand Visual Content with Assistive Technology

- Images should have meaningful alternative text.
- Decorative images should not create unnecessary noise for assistive technology.
- Form fields should have accessible labels.
- Error messages should be understandable.

## US-012: Navigate Using Keyboard

- Interactive elements should be reachable using the keyboard.
- Focus indicators should be visible.
- Users should be able to complete key flows without using a mouse.
- Keyboard navigation order should be logical.
