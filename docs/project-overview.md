# Project Overview

## Project Name

Hotel Booking QA Engineering Project

## Application Under Test

**Application:** Restful Booker Platform / Hotel Booking Demo  
**URL:** https://automationintesting.online/

## Project Description

This project focuses on testing a hotel booking web application from an end-to-end QA perspective.

The application allows users to browse available rooms, review room details, create reservations, submit contact messages, and interact with hotel-related information.

The project is designed to simulate a real QA workflow, starting from test analysis and manual testing, then extending into API testing, UI automation, accessibility, performance, CI/CD, and AI-assisted QA processes.

## Main Business Flows

- View available rooms.
- Select a room.
- Create a booking.
- Validate booking form behavior.
- Submit contact form messages.
- Verify UI behavior across key pages.
- Validate backend/API behavior where applicable.
- Identify accessibility and usability issues.
- Measure basic performance behavior.

## QA Objective

The main objective is to demonstrate a structured QA process that includes:

- Understanding requirements.
- Identifying risks.
- Defining test scope.
- Creating acceptance criteria.
- Designing manual test cases.
- Reporting bugs.
- Automating critical paths.
- Executing API and performance checks.
- Documenting results clearly for technical and non-technical audiences.

## Target Audience

This project is intended for:

- Recruiters.
- QA Leads.
- Engineering Managers.
- Hiring teams looking for Manual QA, QA Automation, or QA Engineer profiles.
